## 2. [content] 系统概览
一句话：本系统由 ESP32-S3 终端、后端服务和管理后台三部分组成，完成从人员管理到设备打卡上传的闭环。
页面结构：三张横向卡片
卡片一：ESP32-S3 终端 —— 摄像头、屏幕、Wi-Fi、本地存储
卡片二：后端服务 —— 认证、接口、数据库、校验
卡片三：管理后台 —— 员工、设备、录脸、考勤
连接说明：管理后台配置，终端本地执行，后端服务校验入库

## 3. [content] 技术组成
一句话：系统三部分分别由以下核心技术支撑，重点对应各自承担的职责。
页面结构：三列纵向排布，按终端 → 后端 → 管理后台
ESP32-S3 终端：ESP32-S3 负责设备运行 · ESP-WHO 完成本地人脸识别 · LVGL 构建设备触摸界面 · SD 卡用于离线缓存
后端服务：设备 HTTP 接口负责同步与上报 · oRPC 提供管理端接口 · Drizzle ORM + libSQL 完成数据持久化 · Better Auth 负责登录认证
管理后台：Nuxt 4 构建浏览器端管理界面 · TailwindCSS 负责页面样式 · oRPC 客户端调用后端业务接口

## 4. [content] 核心功能：人脸识别考勤
一句话：终端本地识别员工身份，根据考勤时间窗口生成打卡记录，并上传到后端服务。
页面结构：左侧设备实拍，右侧 4 步流程
流程：人脸识别 → 员工匹配 → 上下班判定 → 上传入库
补充说明：网络异常时先本地缓存，恢复后自动补传
配图：设备实拍照片、设备屏幕识别/打卡画面

## 5. [content] 首次配置：系统运行前提
一句话：先在管理后台注册并激活终端，再维护组织人员，最后下发录脸任务让终端完成人脸录入。
页面结构：三张截图并列，按配置顺序展示
步骤一：设备注册与激活
步骤二：组织成员建档
步骤三：下发录脸任务并查看录入状态
补充说明：考勤时间窗口在后台统一配置，设备与服务端按同一规则校验
