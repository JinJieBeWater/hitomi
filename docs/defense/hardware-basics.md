# 硬件基础概念速记 🇪🇸 结合本项目

---

## 1. 芯片架构："Xtensa dual-core 32-bit LX7"

**大白话**：ESP32-S3 芯片里有**两个 CPU 核心**（core 0 和 core 1），每个核心都是 32 位的，最高跑 240MHz。

**本项目怎么用**：
```
core 0 ── 重活异步任务（不阻塞界面）
  ├── face_recognition 任务   → 人脸特征提取 + 1:N 比对（~249ms）
  ├── face_enrollment 任务    → 录脸帧处理
  ├── HTTP executor 任务      → 发 HTTP 请求/等响应
  └── WiFi 驱动事件           → 连接/断开回调

core 1 ── 主循环（hitomi_runtime 任务）
  ├── LVGL 界面渲染           → 触摸屏刷新
  ├── 人脸检测（inline）      → MSR+MNP ~38ms，在主 tick 里直接跑
  ├── 摄像头帧轮询 + USB 串口 + 业务逻辑
```

**检测在 core 1（inline），识别在 core 0（异步任务）。**

**记法**：两个大脑——core 0 干重活（识别+网络），core 1 干快活（界面+检测），核间用互斥锁槽位传数据。

## 2. AI 加速：Vector Extension（向量指令）

**大白话**：普通 CPU 一次只能算一个数，但 ESP32-S3 的向量指令可以**一次算 16 个 int8 数**——这就是"加速"。

**类比**：普通快递员一次送一个包裹，向量指令是一次送 16 个——吞吐量大 16 倍。

**具体技术**：128-bit SIMD 寄存器 = 同时放 16 个 int8（8×16=128bit），一条指令完成 16 次乘加。

**本项目怎么用**：
- ESP-DL 的人脸检测和识别模型都用 **int8 量化**
- ESP-NN 的底层卷积、矩阵乘法调用了向量指令
- 同一模型在 ESP32-S3 上比老 ESP32 快 **4.5-6.25 倍**

**记法**：一条指令算 16 个数，int8 量化模型天然适配。

## 3. 存储介质三层

### SRAM（Static RAM）
- **片内**静态内存，速度快（纳秒级访问），容量小
- ESP32-S3 内置 **512KB SRAM**
- 用途：CPU 的"工作台"，放当前正在跑的代码和数据

### PSRAM（Pseudo-Static RAM）
- **片外**（外挂）的大容量内存，通过 SPI 接口连接
- 本项目 SZPI 板：**8MB Octal PSRAM**（OPI = 8 条数据线同时传，比普通 SPI 快很多）
- 访问速度比 SRAM 慢，但容量大 16 倍

**本项目怎么用**：
```
SRAM (512KB)  → 放 LVGL 帧缓冲、小的运行时变量
PSRAM (8MB)   → 放人脸模板（运行时从 SD 卡加载到 PSRAM 做 1:N 比对）
```

**为什么要 PSRAM**：
- 512KB SRAM 不够放人脸模型 + 模板 + LVGL 界面
- 8MB PSRAM 可以放 50 个人的模板（每人 ~2KB 特征向量），识别时直接在内存比对，比每次读 SD 卡快 100+ 倍

**记法**：SRAM = 小工作台（快但小），PSRAM = 大仓库（慢但大，外挂的）。

---

### Flash（闪存）
- **片外**（外挂），断电不丢，容量大，擦写次数有限（~10 万次）
- 本项目：**16MB 外挂 Flash**
- 用途：存固件代码 + 文件系统（SPIFFS / LittleFS）

**"外挂"是什么意思**：
- 芯片本身自带很小的 SRAM
- Flash 和 PSRAM 是额外的芯片，通过引脚焊在 PCB 上
- "外挂" = 不是芯片自带的，是外接的

**本项目怎么用 16MB Flash**：
```
16MB Flash 划分：
├── 固件代码      (~3-4MB)  设备程序本体
├── SPIFFS 分区   (~5-6MB)  人脸检测模型 .bin + 识别模型 .bin（只读）
├── LittleFS 分区  (~2-3MB)  考勤配置、员工快照、待上传队列、失败日志
├── NVS           (~24KB)   设备身份（deviceCode, apiKey）、Wi-Fi 密码
└── OTA 分区等    剩余       空中升级用的
```

**记法**：Flash = 断电不丢的硬盘，存代码 + 模型 + 配置。外挂 = 焊在板子上的额外芯片。

---

### SD Card（SD 卡）
- 可插拔的外部存储，FAT32 文件系统，GB 级容量
- 本项目用 **SDMMC 1-bit 模式**（只用 1 根数据线，不是 4 根，速度够用就行）

**本项目怎么用**：
```
/sdcard/
└── templates/
    ├── manifest.json       ← 模板索引（谁有什么模板）
    ├── emp_001.bin         ← 员工 001 的人脸特征向量
    ├── emp_002.bin
    └── ...
```
- 每个 `.bin` 是 float 数组的二进制，约 ~2KB
- `manifest.json` 记录每个模板的更新时间、大小

**记法**：SD 卡只存人脸模板。Flash 存代码和配置。PSRAM 运⾏时放模板副本用于快速比对。

## 4. BLE（Bluetooth Low Energy）

**大白话**：低功耗蓝牙。和经典蓝牙不一样的是——功耗极低，适合电池设备。

**本项目**：**当前 MVP 不用 BLE**。只用 Wi-Fi。BLE 留作"后续可能用手机 App 配网"的预留能力。

**如果问**："当前只用了 Wi-Fi 通信，BLE 模块硬件支持但暂未启用，留作手机配网的扩展方向。"

## 5. 外设接口（逐个拆解 + 本项目具体使用）

### DVP（Digital Video Port，摄像头接口）
- **并行**数据传输，8 条数据线 + 时钟 + 同步信号
- 专门接摄像头传感器
- 数据格式：YUV422 / RGB565，本项目用 **RGB565**（每个像素 2 字节）

**本项目**：接 OV2640 摄像头模组，最大 200 万像素，实际用 **QVGA (320×240)**。

**为什么不用更高分辨率**：识别模型输入只需 112×112，320×240 够用，帧率更高、内存更省。

**记法**：DVP = 摄像头专用数据通道，8 根线同时传像素。

---

### SPI（Serial Peripheral Interface，串行外设接口）
- 4 根线：MOSI（主机发数据）、MISO（从机发数据）、SCLK（时钟）、CS（片选）
- 全双工，速度快（几 MHz 到几十 MHz）

**本项目怎么用**：
| 外设 | SPI 用途 |
|---|---|
| ST7789 LCD 显示屏 | SPI 发像素数据（240×320，每秒刷屏） |
| PSRAM (OPI) | 8 线 SPI 模式访问大内存 |
| SD 卡 (SPI 模式备选) | 备选（本项目用 SDMMC 而非 SPI） |

**记法**：SPI = 4 根线的快速串行通道，本项目接 LCD 屏幕。

---

### I2C（Inter-Integrated Circuit，两线串行总线）
- 只要 **2 根线**：SDA（数据）+ SCL（时钟）
- 一条总线上可以挂多个设备（每个设备有唯一地址）
- 速度比 SPI 慢（100kHz-400kHz），但省引脚

**本项目怎么用**：
```
I2C 总线上挂了两个设备：
├── FT6336 触摸控制器（地址 0x38）
└── PCA9557 IO 扩展器（控制摄像头电源、LCD 背光等）
```

**记法**：I2C = 两根线的慢速总线，本项目挂触摸屏 + IO 扩展。

---

### SDMMC（SD/MMC 卡接口）
- 专门的 SD 卡/MMC 卡接口
- 支持 1-bit 或 4-bit 模式，比 SPI 模式快
- 本项目用 **1-bit 模式**（CLK + CMD + D0，共 3 根线）

**为什么 1-bit**：考勤终端只需按需读写模板文件，不需要高速连续写入，1-bit 够用且省引脚。

**记法**：SDMMC = SD 卡专用接口，本项目 1-bit 模式存人脸模板。

---

### USB OTG（On-The-Go）
- ESP32-S3 的 USB 可以**同时当设备和主机**
- 当设备：通过 USB 线连电脑，虚拟串口通信
- 当主机：插 U 盘、键盘等

**本项目怎么用**：
- **USB CDC 串口**（当设备）：浏览器通过 Web Serial 经由 USB 线发 JSON 命令配网
- 不用 OTG 主机模式

**记法**：USB OTG = 可当设备也可当主机，本项目只用"当设备"模式做串口配网。

## 速记总表

| 缩写 | 全称 | 几根线 | 快慢 | 本项目接什么 |
|---|---|---|---|---|
| DVP | Digital Video Port | 8 数据 + 控制 | 快 | OV2640 摄像头 |
| SPI | Serial Peripheral Interface | 4 线 | 快 | ST7789 LCD 屏幕 |
| I2C | Inter-Integrated Circuit | 2 线 | 慢 | FT6336 触摸 + IO 扩展 |
| SDMMC | SD/MMC Interface | 3 线 (1-bit) | 中 | SD 卡（人脸模板） |
| USB OTG | USB On-The-Go | USB 线 | — | 电脑 USB 串口配网 |

| 介质 | 断电丢不丢 | 容量 | 速度 | 放什么 |
|---|---|---|---|---|
| SRAM | 丢 | 512KB | 极快 | 运行时变量 |
| PSRAM | 丢 | 8MB | 快 | 人脸模板（运行时副本） |
| Flash | 不丢 | 16MB | 中 | 代码 + 模型 + 配置 |
| SD Card | 不丢 | GB 级 | 慢 | 人脸模板（持久化） |
