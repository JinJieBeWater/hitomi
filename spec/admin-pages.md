# 后台页面设计

## 文档目标

本文档定义当前毕设 MVP 考勤系统后台页面的信息架构、主要展示字段、交互动作和接口依赖。

本文档用于指导以下实现：

- Nuxt 后台页面开发
- 页面组件拆分
- 表单设计
- 列表页和详情交互实现

---

## 适用前提

当前页面设计基于以下前提：

- 单管理员
- 单设备演示场景
- 默认使用 `Asia/Shanghai`
- 不处理员工删除
- 不处理设备删除
- 不处理 `apiKey` 重置
- 后台操作通过 `spec/admin-api.md` 中定义的管理端接口完成

---

## 全局页面约定

### 1. 导航

后台导航至少包含以下页面入口：

- Dashboard
- 员工管理
- 设备管理
- 录脸记录
- 考勤记录

### 2. 表格分页

- 所有列表页默认 `page = 1`
- 所有列表页默认 `pageSize = 20`
- 分页参数与 `spec/admin-api.md` 保持一致

### 3. 时间显示

- 所有页面统一按 `Asia/Shanghai` 展示时间
- 时间展示格式推荐为 `YYYY-MM-DD HH:mm:ss`
- 日期筛选字段使用 `YYYY-MM-DD`

### 4. 状态展示

推荐使用统一中文文案：

| 状态值      | 展示文案 |
| ----------- | -------- |
| `active`    | 启用     |
| `disabled`  | 禁用     |
| `pending`   | 待录入   |
| `success`   | 录入成功 |
| `failed`    | 录入失败 |
| `cancelled` | 已取消   |
| `clock_in`  | 上班     |
| `clock_out` | 下班     |

## 1. Dashboard

### 页面目标

展示系统当前最核心的统计概览，适合答辩演示时快速说明系统状态。

### 数据来源

- `dashboard.summary`

### 页面内容

展示以下内容：

1. 4 张统计卡片
2. 首页内嵌的考勤配置区块
3. 其余常用页面快捷入口

### 展示字段

| 卡片           | 字段                 |
| -------------- | -------------------- |
| 员工总数       | `employeeCount`      |
| 设备总数       | `deviceCount`        |
| 今日上班打卡数 | `todayClockInCount`  |
| 今日下班打卡数 | `todayClockOutCount` |

### 交互动作

- 页面加载时自动请求统计接口
- 提供手动刷新按钮
- 可直接查看并编辑全局考勤配置

### 空状态

- 若系统还没有任何数据，四张卡片均展示 `0`
- 若尚未保存考勤配置，首页考勤配置区块以“首次创建配置”状态展示

## 2. 员工管理页

### 页面目标

维护员工基础信息，并查看员工当前录脸状态。

### 数据来源

- `employee.list`
- `employee.create`
- `employee.update`

### 筛选区域

包含以下筛选项：

- 关键词搜索
- 录脸状态筛选

### 筛选字段

| 字段               | 说明                                                |
| ------------------ | --------------------------------------------------- |
| `keyword`          | 按员工编号或姓名搜索                                |
| `faceProfileState` | `pending`、`success`、`failed`、`cancelled`、`none` |

### 列表字段

表格至少展示：

- 员工编号
- 员工姓名
- 当前录脸状态
- 当前录脸设备名称
- 更新时间
- 操作

### 表格字段映射

| 列名             | 数据来源                      |
| ---------------- | ----------------------------- |
| 员工编号         | `code`                        |
| 员工姓名         | `name`                        |
| 当前录脸状态     | `faceProfile.status` 或空     |
| 当前录脸设备名称 | `faceProfile.deviceName` 或空 |
| 更新时间         | `updatedAt`                   |
| 操作             | 编辑                          |

### 主要动作

- 新增员工
- 编辑员工

### 新增员工表单

字段：

- 员工编号 `code`
- 员工姓名 `name`

校验：

- 两项必填
- 去除首尾空格后不能为空

### 编辑员工表单

字段与新增一致：

- `code`
- `name`

### 页面规则

- 不提供删除功能
- 若员工没有录脸记录，录脸状态显示为空或“未录入”

## 3. 设备管理页

### 页面目标

管理设备基础信息、展示设备码、控制设备启用状态，并提供设备首配入口。

### 数据来源

- `device.list`
- `device.create`
- `device.update`

### 筛选区域

包含以下筛选项：

- 关键词搜索
- 设备状态筛选

### 筛选字段

| 字段      | 说明                   |
| --------- | ---------------------- |
| `keyword` | 按设备名称或设备码搜索 |
| `status`  | `active` 或 `disabled` |

### 列表字段

表格至少展示：

- 设备名称
- 设备码
- 状态
- 最近在线时间
- 创建时间
- 操作

### 表格字段映射

| 列名         | 数据来源        |
| ------------ | --------------- |
| 设备名称     | `name`          |
| 设备码       | `deviceCode`    |
| 状态         | `status`        |
| 最近在线时间 | `lastSeenAt`    |
| 创建时间     | `createdAt`     |
| 操作         | 编辑、启用/禁用 |

### 主要动作

- 创建设备
- 编辑设备名称
- 启用设备
- 禁用设备

### 创建设备表单

字段：

- 设备名称 `name`

### 创建设备成功后的结果展示

必须单独展示以下一次性信息：

- 设备名称
- `deviceCode`
- `bootstrapSerial`
- `bootstrapSecret`

页面要求：

- 明确提示首配需要 `bootstrapSerial` 与 `bootstrapSecret`
- 明确提示运行时 `apiKey` 由设备在激活成功后直接领取，不在管理端展示
- 允许从结果弹窗直接进入串口配置页面

### 编辑设备表单

字段：

- 设备名称 `name`

### 状态切换规则

- `active` 时可执行禁用
- `disabled` 时可执行启用

### 页面规则

- 不提供删除设备功能
- 不提供 `apiKey` 重置功能
- 设备管理页提供“待激活设备”区块
- 设备管理页 header 提供基于 Web Serial 的串口连接入口
- 点击串口连接入口后，浏览器应先弹出串口选择，再尝试连接；连接成功后跳转到独立的串口配置页面，失败则弹出错误提示
- 串口配置页面在读取设备状态后，必须展示当前设备已保存的可编辑配置
- 可编辑配置至少包含后台地址与 Wi-Fi 列表
- Wi-Fi 列表支持动态添加、删除和修改
- 已激活设备也可直接在串口配置页面修改并重新下发当前配置，无需重新创建设备
- 浏览器不支持 Web Serial 时，页面必须给出回退到串口工具的明确指引

## 4. Dashboard 内考勤配置区块

### 区块目标

在 Dashboard 直接维护系统唯一一套全局考勤时间段配置。

### 数据来源

- `attendanceConfig.get`
- `attendanceConfig.save`

### 区块内容

区块包含一个配置表单：

- 上班开始时间
- 上班结束时间
- 下班开始时间
- 下班结束时间

### 表单字段

| 字段              | 说明           |
| ----------------- | -------------- |
| `workStartMinute` | 上班开始分钟数 |
| `workEndMinute`   | 上班结束分钟数 |
| `offStartMinute`  | 下班开始分钟数 |
| `offEndMinute`    | 下班结束分钟数 |

### 前端展示建议

前端表单使用 `HH:mm` 输入控件展示和编辑。

示例默认显示值：

- 上班：`09:00 - 10:00`
- 下班：`18:00 - 19:00`

说明：

- 默认显示值仅用于首次渲染表单
- 后端和数据库不写默认时间

### 校验规则

- 四个字段必填
- 开始时间必须早于结束时间
- 两个时间段不得重叠
- 两个时间段不得相互包含
- 不允许跨天

### 区块动作

- 保存配置

### 区块规则

- 若当前配置不存在，区块以“首次创建配置”状态展示
- 若当前配置已存在，区块以“编辑配置”状态展示

## 5. 录脸记录页

### 页面目标

查看录脸记录，并支持为员工发起、重发或取消待处理录脸任务。

### 数据来源

- `faceProfile.list`
- `faceProfile.enqueue`
- `faceProfile.cancel`
- `employee.list`
- `device.list`

### 筛选区域

包含以下筛选项：

- 状态
- 员工
- 设备

### 筛选字段

| 字段         | 说明                                        |
| ------------ | ------------------------------------------- |
| `status`     | `pending`、`success`、`failed`、`cancelled` |
| `employeeId` | 员工筛选                                    |
| `deviceId`   | 设备筛选                                    |

### 列表字段

表格至少展示：

- 员工编号
- 员工姓名
- 设备名称
- 设备码
- 状态
- 创建时间
- 更新时间
- 操作

### 表格字段映射

| 列名     | 数据来源                 |
| -------- | ------------------------ |
| 员工编号 | `employee.code`          |
| 员工姓名 | `employee.name`          |
| 设备名称 | `device.name`            |
| 设备码   | `device.deviceCode`      |
| 状态     | `status`                 |
| 创建时间 | `createdAt`              |
| 更新时间 | `updatedAt`              |
| 操作     | 创建任务、重新发起、取消 |

### 主要动作

- 新建录脸任务
- 重新发起录脸任务
- 取消待处理录脸任务

### 新建任务表单

字段：

- 员工选择 `employeeId`
- 设备选择 `deviceId`

### 表单规则

- 两项必填
- 设备选择列表仅展示 `active` 设备

### 行级动作规则

- `pending` 状态：显示“取消”
- `failed` 状态：显示“重新发起”
- `cancelled` 状态：显示“重新发起”
- `success` 状态：允许根据需要显示“重新录脸”

### 页面规则

- “重新发起”与“重新录脸”统一调用 `faceProfile.enqueue`
- 当前单设备场景下，设备选择通常只有一项，但页面仍保留设备选择控件

## 6. 考勤记录页

### 页面目标

按日期、员工、设备和类型查看最终有效考勤记录。

### 数据来源

- `attendanceRecord.list`
- `employee.list`
- `device.list`

### 筛选区域

包含以下筛选项：

- 日期
- 员工
- 设备
- 打卡类型

### 筛选字段

| 字段         | 说明                        |
| ------------ | --------------------------- |
| `localDate`  | 单日筛选，格式 `YYYY-MM-DD` |
| `employeeId` | 员工筛选                    |
| `deviceId`   | 设备筛选                    |
| `type`       | `clock_in` 或 `clock_out`   |

### 列表字段

表格至少展示：

- 日期
- 打卡时间
- 打卡类型
- 员工编号
- 员工姓名
- 设备名称

### 表格字段映射

| 列名     | 数据来源        |
| -------- | --------------- |
| 日期     | `localDate`     |
| 打卡时间 | `recognizedAt`  |
| 打卡类型 | `type`          |
| 员工编号 | `employee.code` |
| 员工姓名 | `employee.name` |
| 设备名称 | `device.name`   |

### 页面动作

- 查询
- 翻页

### 页面规则

- 不提供编辑和删除操作
- 列表固定按打卡时间倒序展示
