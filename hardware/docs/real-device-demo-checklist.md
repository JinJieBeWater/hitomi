# 真机端到端演示清单

本文档用于在真实 `SZPI ESP32-S3` 开发板上完成一次可复现的 MVP 演示。

目标闭环：

1. 刷入当前固件
2. 通过后台 Web Serial 页面完成首配
3. 设备完成 bootstrap 激活并拿到正式运行时凭据
4. 设备成功同步后台数据
5. 下发录脸任务并在设备上完成录脸
6. 触发考勤上传并在后台看到结果

---

## 1. 演示前准备

需要满足：

- 开发板已通过 USB 连接到演示机器
- 板载 `SD Card` 已插入
- 设备所在 Wi-Fi 可以访问后台地址
- 浏览器使用 Chromium / Chrome，且允许 Web Serial
- `apps/web/.env` 中的 `APP_BASE_URL` / `BETTER_AUTH_URL` / `CORS_ORIGIN` 指向设备可访问的地址

建议先确认：

- 后台地址不是 `localhost`
- 演示电脑和开发板位于同一局域网
- 浏览器打开后台地址时可以正常登录

---

## 2. 启动后台

在仓库根目录执行：

```bash
bun run dev:web
```

如果只想使用构建后的产物，也可以先构建再预览：

```bash
cd apps/web
bun run build
bun run preview
```

确认点：

- 浏览器可以打开后台首页
- 登录后可进入 `/devices`
- 创建设备、员工、录脸记录页都能正常访问

---

## 3. 烧录固件

在 `hardware/` 目录执行：

```bash
python3 -m platformio run -e szpi_esp32s3 -t upload
```

如需串口日志：

```bash
python3 -m platformio device monitor -b 115200
```

确认点：

- 烧录成功，无校验错误
- 串口日志出现启动信息
- 屏幕点亮，状态页显示 `Activation: setup required` 或待激活状态

---

## 4. 创建设备并保留 bootstrap 凭据

在后台设备管理页：

1. 点击“创建设备”
2. 输入设备名称
3. 保存返回的：
   - `deviceCode`
   - `bootstrapSerial`
   - `bootstrapSecret`

确认点：

- 创建设备成功弹窗出现
- 弹窗可直接跳转到串口配置页

---

## 5. 串口首配

进入 `/devices/serial` 后：

1. 点击“连接设备”
2. 在浏览器串口选择器中选择 ESP32-S3 对应端口
3. 等待页面自动读取 `get_config`
4. 写入：
   - 后台地址 `backendOrigin`
   - 至少一条可用 Wi-Fi
   - 第 4 步创建设备拿到的 `bootstrapSerial` / `bootstrapSecret`

页面预期：

- 能看到当前设备摘要
- 串口日志持续刷新
- 状态从“未配置”进入“待激活”

失败时优先检查：

- 后台地址是否能被设备访问
- Wi-Fi 密码是否正确
- 浏览器是否仍保持串口连接

---

## 6. 激活与同步

在串口配置页点击“写入配置并激活”或“开始激活”后，观察：

- 页面 toast 出现“设备已激活”
- 页面状态变成“已激活”
- 串口日志出现激活请求、成功领取运行时凭据、后续 `/api/device/sync` 调用

后台确认点：

- 设备列表中的目标设备 `lastSeenAt` 刷新
- 设备仍处于 `active`
- 串口配置页能识别到后台中的同名设备

---

## 7. 录脸演示

在后台准备：

1. 创建员工
2. 进入员工页或录脸记录页，为该员工分配当前设备

设备侧预期：

- 同步后拿到 `pending` 录脸任务
- 屏幕进入 Capture / Enrollment 流程
- 提示语从等待人脸进入采样、保存、上报

后台确认点：

- 录脸记录状态从 `pending` 更新为 `success`
- 失败时可在串口日志和状态页看到失败原因

---

## 8. 考勤上传演示

在录脸成功后继续现场识别/打卡，确认：

- 设备侧出现最近一次考勤反馈
- 后台考勤记录页能看到新记录
- `Dashboard` 的今日上班 / 下班计数发生变化

如果没有记录：

- 先检查全局考勤时间段是否覆盖当前时间
- 再检查设备是否已完成最新一次 `/api/device/sync`

---

## 9. 演示后复位

如果要复用同一块板继续演示，可在串口配置页执行：

- 仅清除凭据：保留 Wi-Fi 和后台地址，重新走激活
- 完全重置：清除设备配置，重新进入首配状态

必要时也可重新烧录固件，确保设备回到干净状态。
